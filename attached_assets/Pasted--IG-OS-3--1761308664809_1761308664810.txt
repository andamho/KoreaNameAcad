좋은 세팅인데, 인앱 브라우저(특히 IG/틱톡)가 OS 접근성 배율 + 자체 폰트 부스트를 걸면 위 3가지로도 안 먹히는 경우가 있어요. 아래 순서대로 “안 먹히는 이유 → 즉시 쓰는 패치”를 드릴게요.

왜 아직도 안 되나? (주요 원인)

viewport가 중복/부정확
중복된 <meta name="viewport">나 maximum-scale 등으로 크롬/IG가 Text Autosizing을 재가동.

OS 접근성 배율을 강제 상쇄 불가
인앱 브라우저는 OS의 큰 글자/디스플레이 배율을 우선시합니다. text-size-adjust:100%만으론 못 막음.

제목 계열의 max가 과대
rem을 써도 h1/h2가 넓은 뷰포트에서 과도한 max를 가지면 IG가 줄바꿈/확대를 섞어버립니다.

선언 우선순위/적용 범위
전역에 줬지만 실제로는 컴포넌트 스타일이 더 구체적해서 덮여 있음(특히 Tailwind, CSS-in-JS).

체크리스트 (5분 점검)

뷰포트 단일화

<meta name="viewport" content="width=device-width, initial-scale=1, viewport-fit=cover">
<!-- (maximum/minimum/user-scalable 제거) -->


페이지에 위 태그 하나만 존재하는지 확인(중복 금지).

전역 + 타이포 기본값

html, body{
  -webkit-text-size-adjust: 100% !important;
  text-size-adjust: 100% !important;
}


일부 IG/TT webview가 상속을 잘 안 받아서 body까지 박아두는 게 안전합니다.
필요한 경우 제목 계열에도 개별 부여.

한글 줄바꿈 재설정

/* 한글은 keep-all이 더 자연스럽고 폰트 부스트 트리거를 덜 건드립니다 */
p, h1, h2, h3, h4, h5, h6{
  word-break: keep-all;
  overflow-wrap: anywhere;   /* 영문/URL만 강제 줄바꿈 필요 시 */
  line-height: 1.25;         /* 너무 낮으면 인앱에서 더 키우려 합니다 */
}


지금 break-word가 과하게 줄을 쪼개 IG가 더 키우는 트리거가 되곤 합니다.

제목을 clamp로 ‘보수적’ 제한

.hero-title{
  font-size: clamp(28px, 6.2vw, 40px); /* max를 보수적으로 */
  line-height: 1.15;
}
.hero-sub{
  font-size: clamp(18px, 4.2vw, 24px);
  line-height: 1.4;
}


핵심은 max(우측 값)를 줄이기. 지금 max가 크면 인앱에서 과확대+줄바꿈이 같이 터집니다.

인앱 전용 미세보정(실전용 스니펫)

1) UA 감지 – 반드시 <head> 상단(스타일보다 먼저)

<script>
  (function(){
    var ua = navigator.userAgent || '';
    var root = document.documentElement;
    if (/Instagram/i.test(ua)) root.classList.add('ua-instagram');
    if (/TikTok/i.test(ua) || /Musical\.ly/i.test(ua)) root.classList.add('ua-tiktok');
  })();
</script>


2) 제목/히어로만 낮추는 방어 스타일

html.ua-instagram .hero-title{ font-size: clamp(26px, 5.8vw, 36px); }
html.ua-instagram .hero-sub  { font-size: clamp(17px, 3.8vw, 22px); }

html.ua-tiktok .hero-title{ font-size: clamp(26px, 5.6vw, 36px); }
html.ua-tiktok .hero-sub  { font-size: clamp(17px, 3.6vw, 22px); }

/* 입력 포커스 줌 방지(인앱 확대 트리거 회피: 최소 16px) */
input, select, textarea{ font-size: 16px; }


전역 축소(루트 font-size 하향)는 끝까지 안 되면 1~2%만:
html.ua-instagram{ font-size:15.7px; } (가급적 비권장)

우선순위와 적용 위치(중요)

UA 감지 스크립트는 가능한 한 빨리(FOUC 방지) → 그다음 전용 CSS가 로드되도록.

전역 CSS 뒤에 들어오는 컴포넌트/모듈 CSS가 더 구체적이면 전역이 먹지 않습니다.
⇒ 필요한 곳엔 **클래스 결합(예: .ua-instagram .hero-title)**으로 구체성을 높여 주기.

그래도 안 되면?

OS 접근성(큰 글자/배율) 설정이 큰 기기에서 재현 테스트. 인앱은 이걸 강하게 존중합니다.
→ 이 경우 UA 전용 clamp max 값을 더 낮춰서 타협하는 게 현실적인 해법입니다.

중복 viewport / 다른 라이브러리의 global reset(Tailwind @layer base, styled-components GlobalStyle 등)이
나중에 text-size-adjust를 덮어쓰지 않는지 확인.

OG/링크 미리보기용 캐시(카톡/인스타)는 별도라서, 본문 CSS 반영과 무관하게 썸네일은 늦게 바뀝니다.
